# IWTC-Tools Progress Report – March 5, 2026

## Project Summary

This milestone introduces **graph-based structural indexing and exploratory graph querying**, extending the IWTC indexing pipeline into a navigable representation of the indexed corpus.

Earlier milestones established deterministic indexing of narrative content through a layered pipeline:

raw sources → normalized files → chunks → vocabulary → entity mentions

This phase adds a new structural layer:

entity mentions → graph nodes → graph edges

The graph representation does not introduce new interpretation. Instead, it exposes relationships already present in the indexed data in a form suitable for traversal and structural analysis.

The system now produces several complementary representations of the same corpus:

- **chunked narrative text**
- **deterministic entity mentions**
- **relational lookup indexes**
- **graph-based structural relationships**

Together these layers form a stable foundation for the next stage of development: **semantic enrichment**, where human-curated world relationships can be incorporated into the graph.

At this point the project has transitioned from building indexing infrastructure into developing tools for **structural exploration of the indexed world**.

---

## 1. Index Query Notebook

A new notebook was introduced to make the indexing artifacts directly inspectable.

The notebook allows a human investigator to answer questions such as:

- Where does a given entity appear?
- Which chunks contain those mentions?
- Which files contain those chunks?

The query layer intentionally remains simple and transparent, relying on the canonical index tables rather than constructing new derived structures.

**Key learning:**  
Making the indexes easily inspectable revealed subtle modeling questions earlier than expected, particularly around path handling and entity resolution.

---

## 2. Relational Index Construction

To support efficient querying, several aggregation tables were introduced:

- `index_entity_to_chunks_v0.csv`
- `index_chunk_to_entities_v0.csv`
- `index_player_to_chunks_v0.csv`
- `index_source_files_v0.csv`

These tables provide stable lookup structures derived directly from `ENTITY_MENTIONS_V0` and the chunk metadata.

The purpose of these tables is not to add interpretation but to make common queries efficient and reproducible.

**Key learning:**  
Treating these tables as **derived artifacts rather than primary sources** preserves the authority of the original linking results.

---

## 3. Graph Construction

A new notebook now converts the indexed data into a property graph representation.

The graph currently includes nodes representing:

- Entities
- Vocabulary entries
- Chunks
- Source documents
- Players

Edges represent structural relationships such as:

- `mentions`
- `contains`
- `refers_to`
- `authored_by`
- `cooccurs_with`

This structure allows relationships already implicit in the index to be explored using graph traversal rather than table joins.

**Key learning:**  
The graph layer works best when treated as a **structural view of the index**, not a separate interpretive system.

---

## 4. Graph Artifact Generation

Graph construction now produces the following canonical artifacts:

- `graph_nodes_v0.csv`
- `graph_edges_v0.csv`

These artifacts are designed to be:

- deterministic
- reproducible
- independent of notebook state

They represent the indexed world as a directed property graph suitable for loading into various graph tools.

**Key learning:**  
Separating graph construction from graph querying simplified reasoning about correctness and reproducibility.

---

## 5. Graph Query Notebook

An exploratory graph query notebook was introduced to inspect and experiment with the generated graph.

The notebook supports:

- inspection of node neighborhoods
- examination of evidence chains
- exploratory traversal between entities

Because the current graph is still dominated by evidence relationships, many routes reflect **textual proximity rather than semantic relationships**.

The notebook therefore focuses on **orientation and discovery**, not authoritative interpretation.

**Key learning:**  
Graph traversal becomes significantly more meaningful once curated semantic relationships are introduced.

---

## 6. Graph Structure Observations

Early exploration of the graph revealed several useful structural properties:

- Evidence relationships form a clear spine through:

  `entity → vocabulary → chunk → document`

- Co-occurrence edges provide a lightweight signal of narrative proximity.

- The graph is currently best understood as an **evidence graph**, not yet a semantic world model.

These observations influenced the design of the next development phase.

---

## 7. Current Baseline State

At this checkpoint:

- Raw sources are normalized and chunked
- Vocabulary is curated and applied deterministically
- Entity mentions are linked reproducibly
- Relational index tables support fast lookup
- Graph artifacts represent the indexed corpus as a property graph
- Exploratory graph queries allow structural inspection of relationships

The pipeline now produces **multiple complementary representations of the same indexed corpus**.
