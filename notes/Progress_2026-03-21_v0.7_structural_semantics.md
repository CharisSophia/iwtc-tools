# IWTC-Tools Progress Report – March 21, 2026

## Project Summary

This milestone introduces **semantic graph bootstrapping and structural semantic querying**, transforming the IWTC graph from an evidence model into a **world-aware structural system**.

The pipeline now extends to:

raw sources → normalized files → chunks → vocabulary → entity mentions → graph → semantic relationships

The graph no longer represents only *where things appear*, but *how they are related within the world*.

This establishes **structural semantics**: the ability to identify and rank meaningful relationships between entities using curated world knowledge.

---

## 1. Semantic Relationship Integration

A curated relationship layer (`world_relationships.csv`) was introduced and incorporated into the graph.

- Defines explicit predicates and reverse forms
- Classifies relationships (kinship, political, organizational, etc.)
- Allows controlled inclusion in graph construction

Semantic edges now coexist with evidence edges, enabling traversal across both narrative context and world structure.

**Key learning:**  
Separating evidence from semantics keeps the system interpretable and prevents accidental inference.

---

## 2. Predicate System and Traversal Model

Predicate handling was formalized to support query-time reasoning.

Each predicate now provides:

- forward and reverse forms
- relationship classification
- traversal cost

Cost replaces earlier priority heuristics and defines how “expensive” a relationship is to traverse.

**Key learning:**  
Traversal behavior must be tunable independently of ontology. Cost provides that control.

---

## 3. Path-Based Semantic Querying

Graph querying now centers on **paths between entities**.

The system:

- enumerates valid paths within a cutoff
- renders predicate-aware hops
- supports directed and undirected traversal

This shifts the graph from inspection to **relationship discovery**.

**Key learning:**  
Paths—not nodes—are the core unit of meaning in a semantic graph.

---

## 4. Cost-Based Path Ranking

Path ranking was redesigned around total traversal cost:

- Path cost = sum of edge costs
- Lower-cost paths are preferred
- Direct connections are always prioritized
- Edge count is used as a secondary tie-breaker

This produces stable, predictable ranking behavior across different relationship types.

**Key learning:**  
Summed cost yields more consistent results than per-hop priority comparisons.

---

## 5. Structural Penalties

To prevent misleading paths, penalties were introduced:

- **Kinship penalty** for multi-hop family chains
- **Historical penalty** for former/obsolete relationships

These are applied at the path level to discourage structurally weak explanations.

**Key learning:**  
Correctness depends on path shape, not just edge validity.

---

## 6. Organizational Structure Modeling

The semantic model was expanded to include:

- offices (`off_*`)
- roles (`role_*`)
- command structure (`reports_to`, `supervises`, `holds_office`)

This enables meaningful chains such as:

organization → leadership → operational role → individual

These paths now rank appropriately against simpler but less meaningful alternatives.

**Key learning:**  
Explicit hierarchy modeling outperforms flat membership relationships for reasoning.

---

## 7. Validation and Behavior

Testing across multiple scenarios shows:

- Direct relationships surface correctly
- Shared membership produces strong intermediate paths
- Organizational chains outrank incidental kinship
- Historical relationships are appropriately deprioritized

Path results now align with expected world understanding rather than structural coincidence.

---

## 8. Current Baseline State

At this checkpoint:

- Evidence indexing remains stable and reproducible
- Semantic relationships are integrated into the graph
- Predicate metadata controls traversal behavior
- Path queries return ranked, interpretable relationship chains
- Organizational structures are represented explicitly

The system now supports **structural reasoning over the world model**.

---

## 9. Next Steps

Before beginning narrative semantics, a lightweight incremental rebuild tool will be implemented.

This tool will:
- rebuild indexes and graphs directly to `indexes.path`
- use simple change flags (sources, entities, aliases, relationships, predicates)
- selectively rebuild only affected artifacts (raw indexes, evidence graph, semantic graph)

This addresses the need for rapid iteration when updating entities, relationships, and predicates during ongoing modeling work.

Initial implementation will focus on:
- notebook skeleton
- semantic graph rebuild path

Evidence and raw indexing rebuild paths will be layered in afterward.

Then, the next phase introduces **narrative semantics**: translating structural relationships into human-readable explanations.

Planned work:

1. Evidence object construction (structured path representations)
2. Path interpretation (natural language rendering)
3. Multi-path comparison and explanation ranking
4. Higher-level query abstractions

---

**Current status:**  
The system has transitioned from **semantic enrichment** to **structural semantics**.

Next: **narrative semantics**.
